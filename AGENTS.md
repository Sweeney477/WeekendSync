# WeekendSync Agent Instructions

## Stack

- Next.js 15, React 19, App Router
- Supabase (Auth + Postgres + RLS)
- TypeScript, Zod, Tailwind CSS

## Skills to Use

Apply these skills when relevant to the task:

| Skill | When to Use |
|-------|-------------|
| **nextjs** / **nextjs-weekendsync** | App Router, server components, async params, layout.tsx |
| **nextjs-app-router-patterns** | app/, layouts, Server Components, streaming, data fetching |
| **supabase-postgres-best-practices** | Schema, RLS, migrations, db/, lib/supabase/ |
| **user-journeys** | _project_specs/journeys/, E2E flows |
| **user-onboarding** | Onboarding, sign-in, first-30-seconds |
| **shadcn-ui** | components/ui, forms, dialogs, adding or changing UI components |
| **tailwind-v4-shadcn** | Tailwind theme, globals.css @theme, colors, component styling |
| **accessibility-compliance** | UI components, test:a11y |
| **ux-writing** | Forms, buttons, errors, onboarding copy |
| **playwright-cli** | tests/e2e/ |
| **e2e-testing-patterns** | Writing or debugging E2E tests, flaky tests, critical paths |
| **vitest** | lib/**/*.test.ts |
| **web-design-guidelines** | UI review, accessibility audit |
| **systematic-debugging** | Bugs, test failures, unexpected behavior — find root cause before fixing |
| **verification-before-completion** | Before claiming work done — run tests/checks and confirm output |
| **writing-plans** | Multi-step features — produce implementation plan before coding |
| **frontend-design** / **ui-ux-pro-max** | New pages, UI polish, distinctive design (with web-design-guidelines) |

## Optional Skills

- **security-auditor**: Auth, invite flows, RLS policies
- **interaction-design**: Microinteractions, loading states, polish
- **web-performance-optimization**: Core Web Vitals, bundle size
